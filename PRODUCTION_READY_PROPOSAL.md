# CERT Framework - Production Ready Dashboard Proposal

## Executive Summary

Complete redesign of the dashboard to focus on two core questions from the Anthropic evaluation framework:
1. **Quality Evals**: Is the output good?
2. **Operational Evals**: Can we run this in production?

**Current State Problems:**
- 17 pages with scattered functionality
- Placeholders everywhere (mock data, no real connections)
- Configuration hidden at the bottom of navigation
- No LLM-as-judge capability
- No human review workflow
- Monitoring page shows fake data
- Too ambitious feature set that doesn't work

**New Design Principles:**
- Standalone tool (API-connected, no file uploads as primary flow)
- Production-ready (no placeholders, no demo data)
- Three clear sections + Help
- Configuration FIRST (highlighted, required before using)

### Critique of This Proposal

1. **Configuration might feel like a barrier**
   - Users want to see value before configuring
   - Fix: Add a "Demo Mode" with sample data to explore

2. **Too many clicks to get to key features**
   - Quality â†’ Judge is 2 clicks
   - Fix: Make Judge accessible from Quality overview

3. **Human Review Queue might get ignored**
   - Not visible unless you navigate there
   - Fix: Show badge count in nav when items pending

4. **Missing: Export capabilities**
   - Users need to export reports
   - Fix: Add export button to each dashboard

5. **Missing: Time range selector consistency**
   - Each page has its own time selector
   - Fix: Global time range in header

### Improvements Applied

1. **Sticky notification for pending reviews**
   - Show banner: "12 traces need human review" on all pages

2. **Quick actions on dashboard**
   - "Run Evaluation" button always visible
   - Recent evaluations with one-click re-run

3. **Status indicators in navigation**
   - Green dot = healthy, Yellow = warning, Red = issues
   - Badge count for review queue

4. **Global time range picker**
   - Add to header, affects all pages
   - Options: 1h, 24h, 7d, 30d, Custom

5. **Onboarding flow**
   - First visit â†’ Configuration page
   - Guided setup with progress indicator
   - Skip option for demo mode
